Mission Objective

Connecting Authentication with private CRUD operations.

The Challenge

We built the Auth foundation (Login/Signup). Your goal is to build the "Notes" engine and ensure every user has their own private space that nobody else can access.

Core Goals

  • Protect Note API with JWT Middleware
  • Implement strict User Data Isolation
  • Master Modern Data Fetching (TanStack Query)
  • Build a premium Dashboard UI

Data Architecture

Relational Schemas for Users and their Notes.

User Model

name: String
email: String (unique)
password: Hashed

Note Model

title: String
content: String
user: ObjectId (ref: 'User')
color: String (default: '#fff')

Backend Engineering

Securing the API and enforcing Ownership.

Method Endpoint Requirement Security
GET /api/notes Filter by req.user.id JWT Protected
POST /api/notes Assign user: req.user.id JWT Protected
PUT/DELETE /api/notes/:id Check Ownership before Action JWT Protected

Frontend Integration

Using TanStack Query for a professional experience.

Why TanStack Query?

Say goodbye to useEffect bugs. Use useQuery for fetching and useMutation for updates.

Authorization

Every API call must include the token from LocalStorage in the headers:

Authorization: `Bearer ${token}`

Engineering Standards

Requirements for a Pro-level submission.

Mandatory Rules

  • Strict usage of .env variables
  • Clean Error Handling (Try/Catch)
  • Automatic Logout on 401 Unauthorized
  • Optimistic Updates for better UX

Bonus Challenge

Implement "Pinned Notes" and "Archive" functionality to mirror Google Keep's core features.
